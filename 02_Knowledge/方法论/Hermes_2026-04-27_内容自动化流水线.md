# 内容自动化流水线

**背景**
Jerry的小红书账号"坨坨蔑夫斯基"每日需要发布3篇名画介绍视频，涉及图片生成、TTS配音、视频合成多个环节。Hermes cron任务将全流程自动化。

**核心洞察**
名画介绍视频流水线拆解为4个独立步骤，每步由不同工具完成，最终由ffmpeg合并：
1. **AI图片生成**（mmx）→ 风格锚点锁定（文艺复兴/后印象派/壁画）
2. **TTS配音生成** → 解说词约150字/篇，38-42秒
3. **ffmpeg视频合成** → 1080×1920竖版，钢琴曲背景音乐
4. **飞书通知** → 完成报告发送至会话

各步骤可独立失败、重试，不影响其他步骤。

**对Jerry的意义**
名画系列已形成稳定的"AI图+解说+钢琴BGM"模板，复用率高。每日06:00自动生成3篇视频，08:00和14:00发送，20:04/20:05发布。全程无人值守。

**延伸思考**
- 异常处理：《记忆的永恒》发布时触发平台限流/敏感词检测，需人工介入。可增加重试机制或敏感词预检。
- 图片生成返回格式有时异常（见minimax-cli-workarounds skill），首次生成失败后可直接用上一版输出。
- 视频时长控制：TTS 35-42秒为最佳，太短显得单薄，太长用户流失。
