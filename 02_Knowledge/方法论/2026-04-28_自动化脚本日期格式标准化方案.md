## 问题描述

### 故障现象
```
脚本查找路径: /Volumes/256G/mywork/xhs_content/2026-04-28/post1/
实际路径:     /Volumes/256G/mywork/xhs_content/20260428/post1/
```

脚本执行失败，返回"目录路径格式不匹配"。

### 根本原因
```
# 脚本内写法（BUG）
today = datetime.datetime.now().strftime("%Y-%m-%d")  # → 2026-04-28（有横杠）

# 实际目录结构
/path/to/20260428/post1/  # 无横杠
```

**问题本质**: 代码中硬编码了日期格式假设，而这个假设与运维人员创建目录时使用的手动命名习惯不一致。

## 标准化方案

### 方案一：统一使用无横杠格式（推荐）
```python
# 标准化日期格式常量
DATE_DIR_FORMAT = "%Y%m%d"  # 统一用于所有目录路径
DATE_DISPLAY_FORMAT = "%Y-%m-%d"  # 仅用于日志/报告展示

# 实际使用
today_dir = datetime.datetime.now().strftime("%Y%m%d")  # 用于路径
today_display = datetime.datetime.now().strftime("%Y-%m-%d")  # 用于日志
```

**理由**: 文件系统路径中横杠可能引起问题（如某些命令行工具将其解析为命令行参数），无横杠格式更安全。

### 方案二：配置文件统一管理
```python
import os

# config.py
DATE_DIR_FORMAT = os.environ.get("DATE_DIR_FORMAT", "%Y%m%d")
CONTENT_BASE_PATH = "/Volumes/256G/mywork/xhs_content"

# 使用
today_dir = datetime.datetime.now().strftime(DATE_DIR_FORMAT)
post_path = os.path.join(CONTENT_BASE_PATH, today_dir, "post1")
```

**理由**: 将格式从硬编码提升为可配置，避免修改代码即可适配不同目录结构。

### 方案三：自动探测实际目录
```python
import glob

def find_today_post_dir(base_path, subdir="post1"):
    """自动探测今天的post目录，支持多种日期格式"""
    today = datetime.datetime.now()
    for fmt in ["%Y%m%d", "%Y-%m-%d", "%Y.%m.%d"]:
        try:
            candidate = today.strftime(fmt)
            path = os.path.join(base_path, candidate, subdir)
            if os.path.isdir(path):
                return path
        except:
            pass
    raise FileNotFoundError(f"无法找到今日目录: {base_path}")
```

**理由**: 最灵活的方案，但增加了代码复杂度，适用于历史格式不统一的遗留系统。

## 延伸思考

- **横杠在路径中的隐藏风险**: 在类Unix系统中，横杠 `-` 不是特殊字符，但在与其他工具（如 `find -name "*-*-*"`）配合使用时可能产生歧义。无横杠格式是更安全的选择
- **日志文件名 vs 目录名的格式分离**: 建议约定：目录名用 `YYYYMMDD`，日志文件名用 `YYYY-MM-DD_HH-MM-SS`（保留横杠以增加可读性），两者明确区分用途
- **Windows兼容性问题**: Windows路径对横杠的处理与Unix不同（虽然接受，但某些场景下可能被误解析），无横杠格式在跨平台场景下更鲁棒
