# 渐进式知识积累系统

## 分类：方法论

## 核心理念
> "知识不是整理出来的，是迭代出来的"

---

## 三层架构

### 第一层：输入层（01_Inbox）
**定位：** 收集、记录、不判断
**内容：**
- 笔记（读书/课程）
- 灵感碎片
- 日志（每日复盘）
- 外部输入（播客/论文/对话）

**原则：** 快速记录，不要求结构化

### 第二层：结构层（02_Knowledge）
**定位：** 沉淀、提炼、迭代
**内容：**
- 概念卡片（每个概念一卡）
- 方法论（做事的框架）
- 案例库（成功/失败）
- 项目复盘

**原则：** 必须结构化思考，强制输出

### 第三层：输出层（03_Output）
**定位：** 应用、发布、验证
**内容：**
- 已发布内容（公众号/小红书）
- 汇报材料
- 技能树

**原则：** 知识只有被使用才能迭代

---

## 运作机制

### 日常流
```
输入 → 01_Inbox → 每天/每周整理 → 02_Knowledge
                                   ↓
03_Output ← 输出验证 ← 知识迭代
```

### 复盘驱动
- 每周整理01_Inbox，沉淀到02
- 每次项目结束 → 完整复盘 → 更新02
- 知识只有在输出时才能发现gap

### 版本控制
- Git commit = 每次知识迭代记录
- 不覆盖，只追加/迭代
- 可追溯、可回溯

---

## 工具支撑

### 本地优先
- Obsidian/Logseq 做笔记
- 定期commit到GitHub

### GitHub作为云端备份
- 仓库：gesila_wiki
- 每次commit都是进化节点

### 定时任务自动化
- 日报/周报自动归档到Inbox
- 定期生成内容推送到Output层

---

## 关键规则

1. **输入层不追求完美** — 先记下来再迭代
2. **结构层必须输出** — 教是最好的学
3. **输出层要有反馈** — 没人看的内容等于没写
4. **版本控制 > 完美主义** — 迭代比一次到位更重要
