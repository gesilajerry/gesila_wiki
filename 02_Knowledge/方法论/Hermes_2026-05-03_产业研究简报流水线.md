# 产业研究简报流水线

**背景**
Hermes Cron 驱动的每日产业简报（MLCC产业链晨报、半导体热点晨报、财经新闻推送）已形成稳定范式：MiniMax 联网搜索 → 解析去重 → 飞书 Post 发送。这套流水线的质量取决于搜索关键词设计和结果处理策略。

**核心洞察**

### 第一步：搜索设计
- 使用 `mmx search query --q "关键词1 关键词2 2026" --output json`（空格分隔构成短语匹配，效果更好）
- 每个查询 timeout=30 秒，同时跑 2-3 个关键词再合并去重
- 结果默认按相关性排序，不是时间——如需时间排序，query 中加日期如 "AI新闻 2026年4月"

### 第二步：关键词集合（定向）
| 领域 | 关键词 |
|------|--------|
| MLCC/被动元件 | MLCC、村田、Murata、太阳诱电、Taiyo Yuden、三星电机、国巨、Yageo、华新科、风华高科、三环集团、被动元件 |
| 半导体 | 台积电、ASML、先进封装、CoWoS、2nm、MLCC |
| 财经 | 财经新闻、股市、经济政策、economy stock market news |

### 第三步：结果处理
- 用标题前30字符做去重 key（`key = title[:30]`）
- HTML snippet 清洗：`re.sub(r'<[^>]+>', '', snippet)`，再截取前200字符
- 按相关性取前 8-10 条，时间倒序排列

### 第四步：飞书发送
- 格式：`[[{"tag": "text", "text": "标题：[xxx]\n摘要：[xxx]\n来源：[链接]"}, ...]`
- 每条之间空行分隔
- 文末附"今日关注"一句话趋势点评
- 发送后记录 message_id 便于追溯

**对Jerry的意义**
- 每日简报已跑通，MLCC和半导体晨报已成为稳定的产业跟踪资产
- 搜索质量取决于关键词是否覆盖产业核心玩家（村田/三星电机/国巨等）
- 发现新领域时，只需替换关键词集合即可快速复制流水线

**延伸思考**
当前方案是纯搜索驱动，适合产业动态跟踪。若要深度分析（如竞争格局、扩产跟踪），需结合 IPO 招股书、财报等结构化数据源。
