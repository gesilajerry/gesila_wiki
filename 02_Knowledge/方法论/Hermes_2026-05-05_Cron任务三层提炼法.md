# Hermes Cron任务三层提炼法

**背景**
Hermes每日运行17+ cron任务，产生大量碎片化输出（新闻推送、话题搜索、保障房检查等）。若不处理，这些输出只是噪音而非知识。需要一套将 cron 输出持续转化为可用知识的机制。

**核心洞察**
三层架构解决 cron 任务的知识沉淀问题：
- **输入层（归档）**：cron输出 → 原始文件 100%保留，不删减
- **结构层（提炼）**：原始文件 → 概念卡/方法论卡片，主动提炼
- **输出层（应用）**：结构化知识 → 飞书推送/维基页面，供人使用

三层各自有独立 cron 任务，顺序触发，形成持续运转的知识流水线。

**对Jerry的意义**
不需要手动整理，每次 cron 跑完自动归档+提炼，知识随时间积累。维基知识库成为被动积累的私人知识资产，而非需要主动维护的负担。

**延伸思考**
三层分离的好处：归档层只管保存（不判断价值），结构层专注提炼（不受原始数据干扰），输出层专注分发。三层都可以独立迭代优化，而不影响其他层。
结构层提炼本身也是一个 cron 任务（job_id: 793ce3c39848），意味着提炼动作也被追踪——这是自我追踪能力的体现。
