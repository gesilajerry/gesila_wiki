# 小红书TTS文案长度控制

**背景**
小红书预生成流水线使用 TTS（文字转语音）为视频配音。TTS 时长由文案字数决定，目标区间是 50-90 秒，过短显得单薄，过长会超出平台限制或影响完播率。

**核心洞察**
TTS 语速大约为每分钟 180-200 字（中等语速），对应时长关系：
- 目标时长 50-90 秒 → 文案控制在 **400 字以内**
- 实测案例：蒙克·呐喊文案 496 字 → TTS 111 秒（超出目标）
- 目标案例：克里姆特·吻文案 → TTS 49.3 秒（在下限附近）

经验公式：
```
目标文案字数 ≈ 目标秒数 × 3.5（中速语速）
安全上限：400字（对应约90秒中等语速）
```

**对Jerry的意义**
- 生成小红书文案时，先数一下字数，超过 400 字需要裁剪
- 裁剪优先级：结尾趋势总结 > 中间解释段落 > 开头引子
- TTS 超时的隐含问题：MiniMax TTS 也有时长配额，111 秒的文案消耗更大

**延伸思考**
TTS 时长还受语速设置影响。当前流水线使用默认语速。如需微调，可通过控制文案中的标点密度（逗号会让 AI 停顿）来间接影响节奏，但主要还是字数控制最可靠。
