# 飞书图片上传5MB红线

**背景**
飞书开放平台`im/v1/images`接口上传图片有严格大小限制，超出返回`code 234006`错误。名画原图（如van_gogh_sunflowers.jpg，11.3MB）必须先压缩再上传。

**核心洞察**
飞书API图片上传限制约**5MB**，超过即拒绝。名画高清原图远超此限制，必须压缩后上传。

**对Jerry的意义**
压缩方法（macOS内置，无需安装任何工具）：
```bash
sips -s formatOptions 70 /tmp/xhs_original.jpg --out /tmp/xhs_original_compressed.jpg
```
压缩后约4.5MB，可成功上传。

**注意**：视频文件走`im/v1/files`接口（mp4，限制30MB），图片走`im/v1/images`接口（限制5MB），两套接口两套限额。

**延伸思考**
发送顺序：图片（压缩后）→ 视频 → 文案。每张图独立上传获取image_key，再发送。小红书账号"坨坨蔑夫斯基"三件套发送必须严格遵守此顺序。
