# 日期格式bug导致小红书08:00发布失败

**背景**
2026-04-28 08:00小红书定时发送失败，根本原因是Python日期格式化字符串错误——`%Y-%m-%d`（连字符）vs `%Y%m%d`（纯数字）在路径拼接中的不一致。

**核心洞察**
两个日期格式的混淆：
- `%Y-%m-%d` → `2026-04-28`（ISO格式，含连字符）
- `%Y%m%d` → `20260428`（无连字符）

预生成素材目录用的是无连字符格式（如`/Volumes/256G/mywork/content-assets/images/raw/2026-04-28-post1-img1.jpg`），而脚本中若用`date +%Y%m%d`拼接路径则永远匹配不上。

**对Jerry的意义**
post1目录（08:00）内容实际已就绪，只是脚本找不到文件未能发送。Bug已在post3（20:00）修复并验证成功。

关键代码检查点：
```python
import datetime
today = datetime.datetime.now().strftime("%Y%m%d")  # vs "%Y-%m-%d"
pattern = os.path.join(IMAGES_DIR, f"{today}-post{post_num}-img1.jpg")
```

**延伸思考**
跨系统日期格式不一致是自动化脚本的常见坑。建议在cron任务脚本顶部集中定义日期常量，避免散落在各处重复定义。
