## 🧠 整体架构

```
gesila_wiki/
├── 01_Inbox/              # 📥 输入层（收集）
│   ├── 笔记/              # 读书笔记、课程笔记
│   ├── 灵感/              # 灵感碎片、想法
│   ├── 日志/              # 日复盘、周总结
│   └── 外部输入/          # 播客、论文、对话记录
│
├── 02_Knowledge/           # 📝 结构层（沉淀）
│   ├── 概念卡/            # 每个概念一篇
│   ├── 方法论/            # 做事的框架
│   ├── 案例库/            # 成功/失败案例
│   └── 项目复盘/          # 项目完结后归档
│
├── 03_Output/             # 🚀 输出层（应用）
│   └── 小红书/            # 已发布的小红书文章
│
└── wiki_platform/         # 🌐 知识维基平台（Flask + D3.js）
    ├── app.py            # Flask 主应用
    ├── templates/        # HTML 模板（图谱/卡片/搜索/标签）
    └── static/           # 静态资源（D3.js）
```

## 📝 三层知识框架

### 输入层（01_Inbox）
原始素材收集：读书笔记、灵感碎片、日复盘、外部输入（播客/论文/对话）

### 结构层（02_Knowledge）
沉淀后的结构化知识：
- **概念卡**：一个概念一篇文章
- **方法论**：做事的框架和方法
- **案例库**：成功/失败的真实案例
- **项目复盘**：项目完结后的系统复盘

### 输出层（03_Output）
基于结构层产出的最终内容：小红书文章、汇报材料、技能树

## 🌐 wiki_platform 知识维基平台

本地运行的知识库可视化平台，支持浏览器访问。

**主要功能：**

| 功能 | 说明 |
|------|------|
| 🗺️ **知识图谱** | D3.js 力导向图，可视化三层架构的关联关系 |
| 📄 **卡片浏览** | 按分类浏览所有知识卡片 |
| 🔍 **全文搜索** | 关键词搜索所有卡片 |
| 🏷️ **标签云** | 按标签聚合卡片 |
| 📱 **响应式** | 适配手机/平板/桌面 |

**启动方式：**
```bash
cd /Volumes/256G/gesila_wiki/wiki_platform
python3 app.py
# 访问 http://localhost:5001
# 公网地址在启动日志中查看 Ngrok URL
```

---

## 📌 使用原则

1. **持续迭代** — 每次 commit 都是一次知识进化
2. **结构化输出** — 第二层才是核心价值
3. **定期复盘** — 每周整理输入层，沉淀到结构层
4. **版本控制** — 不覆盖，只追加和迭代
