# gesila_wiki 个人知识库

> 基于"渐进式知识积累"理念，利用GitHub版本控制实现知识的持续迭代与进化。

## 三层知识框架

```
gesila_wiki/
├── 01_Inbox/          # 📥 输入层（收集）
│   ├── 笔记/          # 读书笔记、课程笔记
│   ├── 灵感/          # 灵感碎片、想法
│   ├── 日志/          # 日复盘、周总结
│   └── 外部输入/      # 播客、论文、对话记录
│
├── 02_Knowledge/       # 📝 结构层（沉淀）
│   ├── 概念卡/        # 每个概念一篇
│   ├── 方法论/        # 做事的框架
│   ├── 案例库/        # 成功/失败案例
│   └── 项目复盘/      # 项目完结后归档
│
└── 03_Output/         # 🚀 输出层（应用）
    ├── 已发布/        # 公众号、小红书等
    ├── 汇报材料/       # 方案文档、汇报PPT
    └── 技能树/        # 能力图谱
```

## 使用原则

1. **持续迭代** — 每次commit都是一次知识进化
2. **结构化输出** — 第二层才是核心价值
3. **定期复盘** — 每周整理输入层，沉淀到结构层
4. **版本控制** — 不覆盖，只追加和迭代

## 同步机制

- 本地：`/Volumes/256G/gesila_wiki`
- 远端：`https://github.com/gesilajerry/gesila_wiki`
- 每次知识更新 → commit → push
