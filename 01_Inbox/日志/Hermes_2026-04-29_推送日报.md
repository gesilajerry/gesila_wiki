## OpenClaw Gateway 运维

### 症状（版本 2026.4.24 及之前）
- `openclaw status` / `openclaw update check` 等命令全部超时（30s+）
- `curl http://localhost:18789` 返回 connection refused
- `ps aux | grep openclaw` 显示 gateway 进程在跑但 0% CPU
- 日志 `~/.openclaw/logs/gateway.log` 最后一条是 `"starting channels and sidecars..."` 然后停住
- 日志不写，但 `kill -USR2 <pid>` 可强制刷新日志缓冲区

### 根因（版本 2026.4.24 及之前）
OpenClaw gateway 启动时每次都重新安装所有插件的 bundled runtime deps（npm 包），共 66 个包，在 `~/.openclaw/plugin-runtime-deps/` 目录下。
- 正常需 8-12 分钟
- npm ENOTEMPTY 锁冲突时可达 30 分钟+
- 日志停在 "starting..." 是因为在等插件安装完成，并非真卡住
- 日志文件 600MB+ 会导致日志轮转出问题

### 根因（版本 2026.4.25+）
**browser 插件的 CDP 初始化会卡死 gateway**（0% CPU，端口监听但不响应 HTTP）。
- 进程 PID 存在，端口 18789 监听，但所有 HTTP 请求超时
- 日志不写入（或写入极慢）
- Dev 模式（`OPENCLAW_PROFILE=dev`）因 `skipBootstrap: true` 能起来

**修复**：在 `openclaw.json` 中关闭 browser 插件：
```python
import json
with open('/Users/mac/.openclaw/openclaw.json') as f:
    d = json.load(f)
d['plugins']['entries']['browser'] = {'enabled': False}
with open('/Users/mac/.openclaw/openclaw.json', 'w') as f:
    json.dump(d, f, indent=2)
```

**注意**：如果 browser 插件是你需要的（如 OpenClaw 网页 UI 的浏览器自动化功能），此修复不适用，需进一步排查 CDP 连接问题。

### 诊断步骤
```bash
# 1. 看进程状态
ps -p <pid> -o pid,ppid,state,%cpu,%mem,start,ucomm

# 2. 强制刷新日志看实时进度

# 3. 看 /tmp/openclaw/ 下的实时日志（JSON 格式）

# 4. 检查 npm 锁冲突
```


## 执行模式（在 execute_code 或 terminal 中）

推荐在 `execute_code` 中用 Python subprocess 调用：

```python
import subprocess, json

def mmx_search(query, max_results=10):
    cmd = f'mmx search query --q "{query}" --output json'
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=30)
    data = json.loads(r.stdout)
    results = []
    for item in data.get('organic', [])[:max_results]:
        results.append({
            'title': item.get('title', ''),
            'link': item.get('link', ''),
            'snippet': item.get('snippet', '').replace('</p><p>', ' ').replace('<p>', '').replace('</p>', ''),
            'date': item.get('date', '')
        })
    return results
```

## 搜索多条（多关键词合并去重）

```python
def search_briefing(keywords, max_per_kw=5, max_total=12):
    all_results = {}
    for kw in keywords:
        try:
            results = mmx_search(kw, max_per_kw)
            for r in results:
                # 用 title 做去重 key
                key = r['title'][:30]
                if key not in all_results:
                    all_results[key] = r
        except Exception as e:
            print(f"搜索 '{kw}' 失败: {e}")
    
    items = list(all_results.values())
    items.sort(key=lambda x: x.get('date', ''), reverse=True)
    return items[:max_total]
```


此方式比 gh auth/git-credential 更稳定，OpenClaw 自己的 cron 任务用 gh auth 方式也 push 超时失败。

## OpenClaw Gateway 运维

### 症状（版本 2026.4.24 及之前）
- `openclaw status` / `openclaw update check` 等命令全部超时（30s+）
- `curl http://localhost:18789` 返回 connection refused
- `ps aux | grep openclaw` 显示 gateway 进程在跑但 0% CPU
- 日志 `~/.openclaw/logs/gateway.log` 最后一条是 `"starting channels and sidecars..."` 然后停住
- 日志不写，但 `kill -USR2 <pid>` 可强制刷新日志缓冲区

### 根因（版本 2026.4.24 及之前）
OpenClaw gateway 启动时每次都重新安装所有插件的 bundled runtime deps（npm 包），共 66 个包，在 `~/.openclaw/plugin-runtime-deps/` 目录下。
- 正常需 8-12 分钟
- npm ENOTEMPTY 锁冲突时可达 30 分钟+
- 日志停在 "starting..." 是因为在等插件安装完成，并非真卡住
- 日志文件 600MB+ 会导致日志轮转出问题

### 根因（版本 2026.4.25+）
**browser 插件的 CDP 初始化会卡死 gateway**（0% CPU，端口监听但不响应 HTTP）。
- 进程 PID 存在，端口 18789 监听，但所有 HTTP 请求超时
- 日志不写入（或写入极慢）
- Dev 模式（`OPENCLAW_PROFILE=dev`）因 `skipBootstrap: true` 能起来

**修复**：在 `openclaw.json` 中关闭 browser 插件：
```python
import json
with open('/Users/mac/.openclaw/openclaw.json') as f:
    d = json.load(f)
d['plugins']['entries']['browser'] = {'enabled': False}
with open('/Users/mac/.openclaw/openclaw.json', 'w') as f:
    json.dump(d, f, indent=2)
```

**注意**：如果 browser 插件是你需要的（如 OpenClaw 网页 UI 的浏览器自动化功能），此修复不适用，需进一步排查 CDP 连接问题。

### 诊断步骤
```bash
# 1. 看进程状态
ps -p <pid> -o pid,ppid,state,%cpu,%mem,start,ucomm

# 2. 强制刷新日志看实时进度
kill -USR2 <pid>
tail -20 ~/.openclaw/logs/gateway.log

# 3. 看 /tmp/openclaw/ 下的实时日志（JSON 格式）
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log

# 4. 检查 npm 锁冲突
find ~/.openclaw/plugin-runtime-deps/ -name "*.lock" -delete
```

### 修复方案

**方案 A：清理后重启（推荐）**
```bash
# 清空错误日志（保留文件避免权限问题）
truncate -s 0 ~/.openclaw/logs/gateway.err.log

# 删除 npm 缓存锁
find ~/.openclaw/plugin-runtime-deps/ -name "*.lock" -delete
find ~/.openclaw/plugin-runtime-deps/ -name ".openclaw-npm-cache" -type d -exec rm -rf {} + 2>/dev/null

# 杀掉旧进程
pkill -9 -f "openclaw-gateway"

# 重启
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

**方案 B：跳过飞书 channel 加速启动**
飞书 bot probe 超时（axios 30s × 4 accounts = 2 分钟延迟）：
```python
import json
with open('/Users/mac/.openclaw/openclaw.json') as f:
    d = json.load(f)
d['channels']['feishu']['enabled'] = False
with open('/Users/mac/.openclaw/openclaw.json', 'w') as f:
    json.dump(d, f, indent=2, ensure_ascii=False)
```
重启后 gateway 约 3 分钟可 ready（vs 10 分钟带飞书）。

**方案 C：手动触发 npm 安装（减少重复安装）**
让 gateway 跑完一次后，下次重启会复用缓存。但 npm ENOTEMPTY 问题会反复出现，每次都要清锁。

## 执行模式（在 execute_code 或 terminal 中）

推荐在 `execute_code` 中用 Python subprocess 调用：

```python
import subprocess, json

def mmx_search(query, max_results=10):
    cmd = f'mmx search query --q "{query}" --output json'
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=30)
    data = json.loads(r.stdout)
    results = []
    for item in data.get('organic', [])[:max_results]:
        results.append({
            'title': item.get('title', ''),
            'link': item.get('link', ''),
            'snippet': item.get('snippet', '').replace('</p><p>', ' ').replace('<p>', '').replace('</p>', ''),
            'date': item.get('date', '')
        })
    return results
```

## 搜索多条（多关键词合并去重）

```python
def search_briefing(keywords, max_per_kw=5, max_total=12):
    all_results = {}
    for kw in keywords:
        try:
            results = mmx_search(kw, max_per_kw)
            for r in results:
                # 用 title 做去重 key
                key = r['title'][:30]
                if key not in all_results:
                    all_results[key] = r
        except Exception as e:
            print(f"搜索 '{kw}' 失败: {e}")
    
    items = list(all_results.values())
    items.sort(key=lambda x: x.get('date', ''), reverse=True)
    return items[:max_total]
```

## 飞书 Post 消息发送（踩坑总结）

飞书 post 消息 content 格式极易报 230001 错误（invalid message content），正确格式：

```python
post_content = {
    "zh_cn": {
        "title": "标题",
        "content": [
            [{"tag": "text", "text": "第一行内容"}],
            [{"tag": "text", "text": "第二行内容"}],
            [{"tag": "text", "text": "第三行内容"}],
        ]
    }
}
payload = {
    "receive_id": chat_id,
    "msg_type": "post",
    "content": json.dumps(post_content)  # 必须是 json.dumps 后的字符串
}
```

关键点：
- content 是 array of arrays，每行一个内层数组
- 报错 230001 通常是 content 格式不对，不是 JSON 字符串，或结构错误
- 先用 text 类型消息测试 chat_id 和 token 是否正确

```python
def xfade_pair(a, b, ad, bd, trans=1.5, out):
    offset = ad - trans
    cmd = ['ffmpeg', '-y', '-i', a, '-i', b,
           '-filter_complex', f'[0:v][1:v]xfade=transition=dissolve:duration={trans}:offset={offset:.3f}[outv]',
           '-map', '[outv]', '-an', '-c:v', 'libx264', '-preset', 'fast', '-crf', '22',
           '-t', str(ad + bd - trans), out]
    return subprocess.run(cmd, capture_output=True, text=True)
```

## 常见卡点 & 解法

### 后台进程静默挂起（`.venv/bin/activate` + background=true）
**症状**：进程启动成功（PID 获取），但无任何输出，文件也不生成。
**原因**：login shell（`-l -i -c`）初始化时挂住（等待终端输入），venv activation 被阻塞。
**解法**：
```python
# ❌ 错误：会在 login shell 里挂住
terminal(background=true, command='source /path/to/.venv/bin/activate && python script.py')

# ✅ 正确：直接调 python，venv 通过 sys.path 注入
terminal(background=true, command='/Volumes/256G/mywork/Pixelle-Video/.venv/bin/python /tmp/gen_video.py')
# 或 cd 到目标目录再调 python
terminal(background=true, command='cd /Volumes/256G/mywork/Pixelle-Video && .venv/bin/python /tmp/gen_video.py')
```

### 后台进程有输出但 print 语句被吃掉（输出延迟/无输出）
**症状**：脚本在跑（有 PID，进程存在），但日志文件一直是空的或很久才有输出。
**原因**：Python stdout 默认是 buffered 的，在非交互环境（background=true / subprocess）中不会实时刷新，导致大量输出堆积在缓冲区看不见。
**解法**：所有 `print()` 加 `flush=True`，脚本加 `-u` 参数（unbuffered mode）：
```python
# ✅ 正确：print 加 flush=True
print("开始生成图片...", flush=True)

# ✅ 正确：启动时加 -u 参数
terminal(background=true,
    command='/Volumes/256G/mywork/Pixelle-Video/.venv/bin/python -u /tmp/gen_video.py > /tmp/gen.log 2>&1')

# ✅ 正确：前台跑时也加 -u
terminal(command='/Volumes/256G/mywork/Pixelle-Video/.venv/bin/python -u /tmp/gen_video.py', timeout=600)
```
**调试技巧**：先 `timeout 45` 前台跑一次确认脚本能跑通，再放后台长跑。

```bash

  -vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2"
  -af "volume=1.5" -c:v libx264 -preset fast -crf 22 -c:a aac -b:a 128k -shortest {seg}.mp4

# 两两 xfade（dissolve 交叉淡化，比 filter_complex 链更可靠）
ffmpeg -y -i {a}.mp4 -i {b}.mp4
  -filter_complex "[0:v][1:v]xfade=transition=dissolve:duration=1:offset={dur_a-1}[outv]"
  -map [outv] -an -c:v libx264 -preset fast -crf 22 -t {dur_a+dur_b-1} {out}.mp4
```

### 已知坑（踩过）

1. **mmx `--output` 只支持目录**：`--output /dir/` 保存到 `/dir/{timestamp}.mp3`，不支持 `--output /dir/file.mp3`，需手动 copy
2. **mmx image 输出自动加 `_001`**：`--out-prefix foo` → `foo_001.jpg`，不是 `foo.jpg`
3. **ffmpeg `-shortest` 导致音视频被截断**：在 video+audio 合并时慎用，换成指定 `-t DURATION`
4. **字幕流需要 `-map 3:s`**：`-i srt` 后不自动烧入，需要明确 `-map 3:s -c:s mov_text`
5. **xfade offset 计算**：`offset = 前段时长 - TRANSITION`，xfade 从前段末尾往前 TRANS 秒开始
6. **钢琴配乐循环**：钢琴曲约 3 分钟，文案约 1.5-2 分钟，不需要循环，直接淡出即可
7. **飞书视频 20MB 限制**：最终视频需压缩 `ffmpeg -crf 26-28 -vf scale=720:-2 -c:a aac -b:a 64k`

### `--output` flag is actually a directory, not a full path
`mmx speech synthesize --output /path/to/file.mp3` does NOT save to that path. The file is saved with an auto-generated name in CWD (e.g., `speech_2026-04-26-02-10-10.mp3`). The `--output` argument is misinterpreted.

**Workaround**: Use `--output .` (current directory) and read the `saved` field from JSON output, then copy/rename manually.

```python
import re

def md_to_html(md_text):
    """不含 emoji 过滤（Python 3.14 regex bug 会损坏 CJK 字符）"""
    html = md_text
    html = re.sub(r'^---\n[\s\S]+?\n---\n', '', html)
    html = re.sub(r'^# (.+)$', '<h1>\\1</h1>', html, flags=re.MULTILINE)
    html = re.sub(r'^## (.+)$', '<h2>\\1</h2>', html, flags=re.MULTILINE)
    html = re.sub(r'^### (.+)$', '<h3>\\1</h3>', html, flags=re.MULTILINE)
    html = re.sub(r'^#### (.+)$', '<h4>\\1</h4>', html, flags=re.MULTILINE)
    html = re.sub(r'^---$', '<hr>', html, flags=re.MULTILINE)
    html = re.sub(r'```(?:\w*)\n([\s\S]*?)\n```', '<pre><code>\\1</code></pre>', html)
    html = re.sub(r'`([^`]+)`', '<code>\\1</code>', html)
    html = re.sub(r'\*\*(.+?)\*\*', '<strong>\\1</strong>', html)
    html = re.sub(r'\*(.+?)\*', '<em>\\1</em>', html)
    html = re.sub(r'^> (.+)$', '<p><em>\\1</em></p>', html, flags=re.MULTILINE)
    # × 符号可能导致 500，标题级别在调用处处理
    lines = html.split('\n')
    in_table = False
    new_lines = []
    for line in lines:
        if '|' in line and line.strip().startswith('|'):
            cells = [c.strip() for c in line.strip().strip('|').split('|')]
            if not in_table:
                new_lines.append('<table border="1" cellpadding="8" cellspacing="0">')
                in_table = True
            is_header = all(re.match(r'[-:]+$', c.replace(' ', '')) for c in cells)
            tag = 'th' if is_header else 'td'
            new_lines.append('<tr>' + ''.join(
                f'<{tag}>{c}</{tag}>' for c in cells
                if not re.match(r'^[-:]+$', c.replace(' ', ''))
            ) + '</tr>')
        else:
            if in_table:
                new_lines.append('</table>')
                in_table = False
            new_lines.append(line)
    if in_table:
        new_lines.append('</table>')
    return '\n'.join(new_lines)

def extract_title(md):
    """提取标题并处理特殊字符"""
    m = re.search(r'^#\s+(.+)$', md, re.MULTILINE)
    title = m.group(1).strip() if m else '无标题'
    title = title.replace('×', 'x')  # × 会导致服务端 500
    return title
```

## 执行模式（在 execute_code 或 terminal 中）

推荐在 `execute_code` 中用 Python subprocess 调用：

```python
import subprocess, json

def mmx_search(query, max_results=10):
    cmd = f'mmx search query --q "{query}" --output json'
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=30)
    data = json.loads(r.stdout)
    results = []
    for item in data.get('organic', [])[:max_results]:
        results.append({
            'title': item.get('title', ''),
            'link': item.get('link', ''),
            'snippet': item.get('snippet', '').replace('</p><p>', ' ').replace('<p>', '').replace('</p>', ''),
            'date': item.get('date', '')
        })
    return results
```

## 搜索多条（多关键词合并去重）

```python
def search_briefing(keywords, max_per_kw=5, max_total=12):
    all_results = {}
    for kw in keywords:
        try:
            results = mmx_search(kw, max_per_kw)
            for r in results:
                # 用 title 做去重 key
                key = r['title'][:30]
                if key not in all_results:
                    all_results[key] = r
        except Exception as e:
            print(f"搜索 '{kw}' 失败: {e}")
    
    items = list(all_results.values())
    items.sort(key=lambda x: x.get('date', ''), reverse=True)
    return items[:max_total]
```

## ⚠️ multipart/form-data 构造常见错误

用 urllib 手动拼接时，禁止在非文件字段加 `filename=`，否则 Feishu 返回 400。

❌ 错误（把 file_name/image_type 当文件发送）：
```
--{boundary}\r\nContent-Disposition: form-data; name="file_name"; filename="video.mp4"\r\n
```

✅ 正确（用 requests 分离 files + data）：
```python
# 视频上传
files = {"file": (fname, f, "video/mp4")}
data = {"file_name": fname, "file_type": "mp4"}
r = requests.post(url, headers={"Authorization": f"Bearer {token}"}, files=files, data=data, timeout=120)

# 图片上传同理，image_type 不要放 filename 里
files = {"image": (fname, f, "image/jpeg")}
data = {"image_type": "message"}
r = requests.post(url, headers={"Authorization": f"Bearer {token}"}, files=files, data=data, timeout=30)
```

## ⚠️ multipart/form-data 构造常见错误

用 urllib 手动拼接时，禁止在非文件字段加 `filename=`，否则 Feishu 返回 400。

❌ 错误（把 file_name/image_type 当文件发送）：
```
--{boundary}\r\nContent-Disposition: form-data; name="file_name"; filename="video.mp4"\r\n
```

✅ 正确（用 requests 分离 files + data）：
```python
# 视频上传
files = {"file": (fname, f, "video/mp4")}
data = {"file_name": fname, "file_type": "mp4"}
r = requests.post(url, headers={"Authorization": f"Bearer {token}"}, files=files, data=data, timeout=120)

# 图片上传同理，image_type 不要放 filename 里
files = {"image": (fname, f, "image/jpeg")}
data = {"image_type": "message"}
r = requests.post(url, headers={"Authorization": f"Bearer {token}"}, files=files, data=data, timeout=30)
```

## 执行模式（在 execute_code 或 terminal 中）

推荐在 `execute_code` 中用 Python subprocess 调用：

```python
import subprocess, json

def mmx_search(query, max_results=10):
    cmd = f'mmx search query --q "{query}" --output json'
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=30)
    data = json.loads(r.stdout)
    results = []
    for item in data.get('organic', [])[:max_results]:
        results.append({
            'title': item.get('title', ''),
            'link': item.get('link', ''),
            'snippet': item.get('snippet', '').replace('</p><p>', ' ').replace('<p>', '').replace('</p>', ''),
            'date': item.get('date', '')
        })
    return results
```

## 搜索多条（多关键词合并去重）

```python
def search_briefing(keywords, max_per_kw=5, max_total=12):
    all_results = {}
    for kw in keywords:
        try:
            results = mmx_search(kw, max_per_kw)
            for r in results:
                # 用 title 做去重 key

```
