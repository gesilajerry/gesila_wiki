xfade 拼接时必须加 `-an`（去掉音频），因为两个片段的音频时长不同、采样率不同，直接拼接会有杂音。**正确的顺序是：**
1. xfade 拼接视频（-an）
2. 最后一步统一混音（配音原音频 + 背景音乐）

### mmx 文件名后缀
`mmx image generate` 保存文件时会在后面加 `_001` 后缀：
```
seg9_img_5a.jpg  →  seg9_img_5a_001.jpg
```
生成后立即重命名：
```bash
for f in seg9_img_*.jpg; do new=$(echo $f | sed 's/_001\.jpg/.jpg/'); mv "$f" "$new"; done
```

### subprocess 传参要用 list
Python `subprocess.run()` 第一个参数**必须是 list**，不能是 string：
```python
# ✅ 正确
subprocess.run(['ffmpeg', '-y', '-i', tts_file, '-ss', str(start), '-t', str(dur), '-c', 'copy', tmp_tts],
               capture_output=True, text=True, timeout=30)
# ❌ 错误（FileNotFoundError）
subprocess.run('ffmpeg -y -i %s ...' % args, shell=True, ...)
```

ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
TOKEN=*** Token 上文"

def publish_one(title, content_html, group_ids, attempt=1):
    """发布单篇文章，每次单独获取 CSRF"""
    try:
        # 每次单独获取 CSRF，不要复用
        csrf_req = urllib.request.Request(
            'https://192.168.30.198:8443/api/csrf',
            headers={'Authorization': f'Bearer {TOKEN}'})
        csrf = json.loads(
            urllib.request.urlopen(csrf_req, context=ctx, timeout=10).read()
        )['data']

        payload = json.dumps({
            "forum_id": FORUM_ID,
            "title": title,
            "content": content_html,
            "group_ids": group_ids,
            "type": "blog"
        }, ensure_ascii=False).encode('utf-8')  # ensure_ascii=False 保存中文

        req = urllib.request.Request(
            'https://192.168.30.198:8443/api/discussion',
            data=payload,
            headers={
                'Authorization': f'Bearer {TOKEN}',
                'X-CSRF-Token': csrf,
                'Content-Type': 'application/json; charset=utf-8',
                'Accept': 'application/json'
            },
            method='POST')
        return json.loads(urllib.request.urlopen(req, context=ctx, timeout=15).read())

    except urllib.error.HTTPError as e:
        if e.code == 500 and attempt < 3:
            time.sleep(3)
            return publish_one(title, content_html, group_ids, attempt + 1)
        return {'success': False, 'error': f'HTTP {e.code}: {e.reason}'}

### Step 2: Markdown 转 HTML

```python
import re

def md_to_html(md_text):
    """不含 emoji 过滤（Python 3.14 regex bug 会损坏 CJK 字符）"""
    html = md_text
    html = re.sub(r'^---\n[\s\S]+?\n---\n', '', html)
    html = re.sub(r'^# (.+)$', '<h1>\\1</h1>', html, flags=re.MULTILINE)
    html = re.sub(r'^## (.+)$', '<h2>\\1</h2>', html, flags=re.MULTILINE)
    html = re.sub(r'^### (.+)$', '<h3>\\1</h3>', html, flags=re.MULTILINE)
    html = re.sub(r'^#### (.+)$', '<h4>\\1</h4>', html, flags=re.MULTILINE)
    html = re.sub(r'^---$', '<hr>', html, flags=re.MULTILINE)
    html = re.sub(r'```(?:\w*)\n([\s\S]*?)\n```', '<pre><code>\\1</code></pre>', html)
    html = re.sub(r'`([^`]+)`', '<code>\\1</code>', html)
    html = re.sub(r'\*\*(.+?)\*\*', '<strong>\\1</strong>', html)
    html = re.sub(r'\*(.+?)\*', '<em>\\1</em>', html)
    html = re.sub(r'^> (.+)$', '<p><em>\\1</em></p>', html, flags=re.MULTILINE)
    # × 符号可能导致 500，标题级别在调用处处理
    lines = html.split('\n')
    in_table = False
    new_lines = []
    for line in lines:
        if '|' in line and line.strip().startswith('|'):
            cells = [c.strip() for c in line.strip().strip('|').split('|')]
            if not in_table:
                new_lines.append('<table border="1" cellpadding="8" cellspacing="0">')
                in_table = True
            is_header = all(re.match(r'[-:]+$', c.replace(' ', '')) for c in cells)
            tag = 'th' if is_header else 'td'
            new_lines.append('<tr>' + ''.join(
                f'<{tag}>{c}</{tag}>' for c in cells
                if not re.match(r'^[-:]+$', c.replace(' ', ''))
            ) + '</tr>')
        else:
            if in_table:
                new_lines.append('</table>')
                in_table = False
            new_lines.append(line)
    if in_table:
        new_lines.append('</table>')
    return '\n'.join(new_lines)

def extract_title(md):
    """提取标题并处理特殊字符"""
    m = re.search(r'^#\s+(.+)$', md, re.MULTILINE)
    title = m.group(1).strip() if m else '无标题'
    title = title.replace('×', 'x')  # × 会导致服务端 500
    return title
```

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 401 unauthorized | CSRF Token 过期 | 重新从 GET /api/csrf 获取 |
| csrf token mismatch | CSRF Token 为空字符串 | 从响应 `data` 字段提取实际值 |
| 发布成功但列表查不到 | 可能是 group_ids 权限问题 | 确认账号有该分类权限 |
| koala_session httpOnly | 浏览器 cookie 是 httpOnly | 使用 Bearer Token 而非 cookie |
| HTTP 500 随机出现 | CSRF Token 复用或服务端波动 | 每次单独获取 CSRF + 3次重试 |
| HTTP 500 稳定复现（标题含×） | `×`(U+00D7) 导致 JSON 解析失败 | 将 `×` 替换为 `x` 后重试 |
| HTML 变乱码（CJK被删） | Python 3.14 emoji 正则 bug 误删 CJK | 不要用 emoji 过滤函数，原文直传 |
| HTTP 500 err:ratelimit | 请求过于频繁，触发服务端限速 | 间隔5秒以上再重试，批量发布时每篇间隔1.5-2秒 |

## Git Push 凭证问题（HTTP/SSH 混合环境）

**问题**：git push 时一直超时或 `could not read Username`——credential helper（osxkeychain / `gh auth git-credential`）在这个环境里读不到 token，导致无法认证。

**原因**：macOS 钥匙串里的 GitHub PAT 没被 git 正确调用，HTTPS Push 无法弹出认证。

**解决方案：创建 netrc 文件**

```python
import subprocess, os
token = subprocess.run(['gh', 'auth', 'token'], capture_output=True, text=True).stdout.strip()
home = os.path.expanduser('~')
with open(f'{home}/.netrc', 'w') as f:
    f.write(f"""machine github.com
login gesilajerry
password {token}
""")
os.chmod(f'{home}/.netrc', 0o600)
```

然后在 git push 时指定使用 store helper：

```bash
GIT_TERMINAL_PROMPT=0 git -c credential.helper=store push origin main
```

**验证**：`~/.netrc` 权限必须是 600。
