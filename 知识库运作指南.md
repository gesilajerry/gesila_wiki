# gesila_wiki 知识库运作指南

> Jerry的个人知识积累系统 | 2026-04-12建立

---

## 一、核心原则

### 输入层（01_Inbox）— 100%保留，不删减
- **来源：** 定时任务推送 / Jerry发送 / 网络搜索 / 文档阅读
- **原则：** 原样保存，不简化、不摘要、不提炼
- **格式：** Markdown文件，文件名带时间戳
- **分类：**
  - `日志/` — 每日定时任务原始内容
  - `笔记/` — 读书/课程/会议笔记
  - `灵感/` — 灵感碎片、想法
  - `外部输入/` — 播客、论文、文章

### 结构层（02_Knowledge）— 提炼、迭代、卡片化
- **来源：** 输入层定期整理提炼
- **原则：** 结构化思考，一个概念一卡片
- **分类：**
  - `概念卡/` — 核心概念定义和理解
  - `方法论/` — 做事的框架和流程
  - `案例库/` — 成功/失败案例详细记录
  - `项目复盘/` — 项目完结后完整复盘

### 输出层（03_Output）— 应用、发布、反馈
- **来源：** 结构层知识应用
- **原则：** 知识只有被使用才能迭代
- **分类：**
  - `已发布/` — 公众号、小红书等已发布内容
  - `汇报材料/` — 方案文档、工作汇报
  - `技能树/` — 能力图谱

---

## 二、运作流程

### 日常流程
```
Jerry发送素材 / 定时任务推送
         ↓
保存到01_Inbox对应分类（100%保留原文）
         ↓
每日cron任务自动整理归档
         ↓
定期提炼到02_Knowledge（每周或项目结束）
         ↓
输出到03_Output并发布
         ↓
Git commit + push到GitHub
```

### 触发机制
1. **Jerry直接发送** — "帮我保存这个" → 直接归档
2. **Jerry要求读取/搜索** — 搜索结果保存到Inbox
3. **定时任务推送** — 每日2次自动归档到日志
4. **定期整理** — 周末结构层提炼
